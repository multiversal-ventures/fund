Data Center Adjacency — Commercial Property Thesis

Position around the data center build-out through land banking, conversion of industrial/commercial stock, and adjacency to clusters — not operating colocation or hyperscale facilities. National screen with emphasis on emerging markets where power and politics align before commercial property fully reprices.

Tier 1 model (live): County-level dc_market_score (0–100) is computed in scripts/dc/, stored as data/dc/dc_market_scores.parquet, and loaded in Fund Explorer → DC Markets. Percentiles are computed only on eligible counties (dc_eligible = true): min population 50k, min housing units 20k, and island territories excluded (PR, VI, GU, AS, MP). Ineligible rows are kept for joins but have null scores. Water/connectivity layers use neutral placeholders until Phase 2; electrical, pipeline, and political signals drive most dispersion among eligibles.

Methodology — how we score

Each metric is percentile-ranked across counties, then multiplied by category weights (see scripts/dc/dc_weights.default.json). Weights align with third-party frameworks: combined electrical + water/cooling ≈ 40% (compare Kearney and Analytics.loan power-heavy models); pipeline is intentionally lower than Analytics.loan’s 30% pipeline pillar so we do not only chase already-hot markets.

Category weights (100 pts before penalties)

  • Electrical (30) — State industrial ¢/kWh (bundled EIA-style CSV in-repo; replace with EIA API when wired). Lower cost = better.
  • Water & cooling (10) — Placeholder constant until USGS / climate layers land (neutral spread).
  • Political (20) — Tavily Search API, state-level queries for incentives vs. risk language; see Tavily Agent Skills and docs index.
  • Pipeline (15) — Census CBP NAICS 518210 employment per 1k population (county).
  • Connectivity (10) — Placeholder until FCC / IX data merged.
  • Labor & cost (10) — Workforce resilience index from occupations_2023 when present.
  • Unique (5) — Placeholder for renewable / disaster subscores.
  • Penalties — Tavily-derived moratorium/opposition signal (capped).

Run the pipeline

From repository scripts/ (with Census ACS already pulled to data/census/acs_2023.parquet):

uv run python dc/run_dc_pipeline.py --skip-tavily

Omit --skip-tavily when TAVILY_API_KEY is set. Upload data/dc/*.parquet with your existing Firebase upload flow (uv run run.py --upload after files are in data/).
